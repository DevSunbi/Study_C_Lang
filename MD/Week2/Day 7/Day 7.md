## 포인터 배열
#### 포인터 배열 선언과 사용
* 포인터 배열과 배열의 포인터 헷갈릴 수 있음
* 포인터와 배열은 동격이라 함 -> 컴파일러 내부에서는 차이가 존재
* 배열로 봄 -> base size 및 size가 정해져 있음
* 포인터 변수로 봄 -> base size는 동일, 배열의 경계 체크 방법이 없음
* 2차원 배열로 가는 순간 문제 발생
* 포인터를 더블 포인터로 선언해야하냐, 싱글 포인터로 선언해야하나
	* 표기법에 있어 혼동 발생
* char \*pary[5]의 우선순위는 누구?
	* 포인터 변수들이 5개의 배열로 있다
	* 배열을 가리키는 포인터 변수다 -> 포인터 변수에 괄호 쳐줘야함
	* char\*의 포인터 변수가 5개
* 포인터 배열 초기화
	* char \*pary[5] = {"...", "...", ...} => 상수(문자열의 시작 주소만 배열 요소에 저장)
	* char animals\[5]\[20] = {"...", "...", ...} => 변수(문자열 자체를 배열의 공간에 저장)
* base size 모르면 첨자 사용한 주소 연산 엄금
* pary\[2]\[2] == \*(pary[2] + 2)
* 주소 계산과 간접 참조 연산으로 각 위치 찾아감

## 이중 포인터와 배열 포인터
#### 이중 포인터 개념
* 포인터 변수 사이즈 만큼이 포인터 크기
* 포인터 변수의 주소값을 가짐
* 포인터가 가리키는 자료형과 포인터 자체의 형태가 다름
* (double \*\*)형 이중 포인트는 (double \*)형 포인터를 가리킴
* 2차원 배열 더블 포인터로 받으면 안됨 -> double pointer의 base type과 2차원 배열 base type 해석 방법이 다름
* &로 주소를 전달하나 그냥 배열의 요소를 전달하나 결국은 시작 주소를 보냄에 차이가 없음
	* 다만 연산 시 차이가 존재하니 조심
#### 배열 포인터
* type (\*val)\[n] : type형 변수 n개의 배열을 가리키는 배열 포인터
	* base type이 type인 n개 배열을 만남
* 다차원 배열의 경우 base size가 전달 되어야 성립됨
* int (\*pa)\[4] -> int 4 개의 1차원 배열을 가리킴
#### 함수 포인터
* func는 code segment에 올라감 -> 메모리 특정 영역에 올라감
* 함수 호출 코드 만남 -> 다음번 실행 문장이 해당 함수, 이후 리턴
* 결국 함수 실행은 함수 시작 메모리로 이동해 실행한다와 동일한 얘기
* 원래 함수 이름 -> 함수의 주소 상수 => 포인터에 저장 가능
* n개의 type을 받아서 type을 리턴하는 함수의 주소를 담은 포인터
* 특정한 이름으로 호출 했는데 연계해 간접 호출 시 사용
* 이벤트 핸들러 등에서 자주 사용
#### void 포인터
* 정수 포인터로 바꿔야지 해석 가능
* 어떤 타입의 데이터든 담을 수 있음
	* 형변환 필수
* 간접 참조 연산과 주소에 대한 정수 연산이 불가
* 임의의 주소 저장 가능
* 가리키는 자료형에 대한 정보 없음
## 동적 할당 함수
#### malloc, free
* 메모리 공간은 4 section(data segment(Data, BSS, ROData), heap, stak, Code segment)
* stack은 함수(변수 연산) 종료까지 메모리 할당 유지 종료 시 해제
* 컴퓨터 -> 메모리라는 자원을 memory management system을 통해 관리(OS)
	* 공간 할당 해제하며 자동화된 관리 유지
	* 자원을 쌓아놓고 요청 시 할당(이후 자동 해제)
	* heap 메모리 공간은 수동으로 유저가 할당 받음 -> 해제는 필수(수동)
	* 해제 안하면 메모리 누수 발생 -> 가용 메모리의 감소, 이후 메모리 할당 문제 발생
* (datatype \*)malloc(sizeof(datatype) * n) 형태
	* 해제 안하면 잘못된 공간에 진입하는 것이 아닌 메모리 공간 자체를 낭비하는 행위
	* 메모리를 할당해준다 -> 포인터/배열과 동일하게 생각하면 편함
	* void pointer로 돌려 받은 후 casting, 상기 목적에 맞게 작업 진행
	* 주소를 공간하는 공간이지 데이터를 저장하진 않음
* exit 함수도 종료 함수 -> return은 종료 키워드 => exit(1)은 1을 반환함을 의미
	* exit(0)는 0리턴, 정상 종료, 1은 비정상 종료를 의미 / 그 외는 에러 코드 취급
* malloc 함수는 첫 번째 시작 주소를 던져줌, 포인터 사용이 필수
	* 배열의 이름은 상수 주소이기에 이미 본인의 주소 존재, 사용 불가
	* 포인터 변수에 malloc으로 주소 할당 -> 배열과 동일한 문법 사용 가능(\[])
* free -> 메모리 할당 해제 함수
	* 시작 주소를 전달, 할당 메모리 해제 후 OS에 전달(해제 여부)
	* malloc 없이 free를 사용하면 에러 발생(애시당초 할당이 안되어 있음)
	* free는 단 한번만 사용, 이후 사용은 안됨
	* Null 값을 지정해주는 형식
		* free해도 포인터 변수에 주소값 존재, 댕글링 포인터 재접근 막기 위해 NULL 초기화
* 힙 메모리의 여유 공간 없을 시 할당 실패 경우 존재, 조건문을 활용해 NULL인지 여부 확인
* free에 몇 바이트의 메모리 공간을 해지하는지 명시되어 있는 내용 없음
	* malloc에서 할당 시 헤더가 붙음, 헤더에 명시되어 있는 사이즈 정보를 읽은 후 확정
#### calloc, realloc
* calloc -> 할당한 공간을 0으로 초기화
	* void \*calloc(unsigned int, unsigned int)
* realloc -> 할당한 공간의 크기를 늘이거나 줄임
	* void \*realloc(void \*, unsigned int)
	* 현재 위치 바로 뒤에 여유 공간이 없으면 새 메모리 할당 후 데이터 복사
		* 이 때 기존 메모리는 자동 free 진행 이후 새 주소 리턴
* 그럼 메모리를 합쳐보자(메모리 압축)
	* 합친다 -> 메모리 공간의 이동(복사 및 새 할당 필요) -> 주소값 변동 -> 프로그램 실행 중엔 제대론 된 값에 접근 못하는 이상현상 발생 가능성 존재 => 함부로 못함
	* 오버헤드 상당히 큼
	* 해당 작업 수행 시 프로그램 일시 정지 후 재실행
	* 해당 현상 방지를 위한 메모리 풀 사용
		* 사용할 크기 미리 정함, 해당 범위 안에서 할당, 해제 반복(조각 안나게 관리)
		* 진짜 방지하려면 정적 할당 하면 됨(malloc말고 전역변수로 크게 잡고 사용)
			* => 개 푸씨 겁쟁이클럽 기법
