### Introduction

I2C 통신은 MCU와 주변 장치 사이의 통신에 사용 가능한 통신 방식

짧은 거리에서 빠르지 않은 속도로 데이터 전송이 필요할 때 주로 사용

I2C 통신은 필립스(지금은 NXP)에 의해 TV 내부 회로에서 사용하기 위해 개발된 통신 방식

대부분의 MCU에 내장이 될 만큼 주변 기기와의 통신에 많이 사용

속도가 빠르진 않지만 2개의 선으로 통신 가능, 버스 방식으로 한 개의 포트에 여러 개의 장치 붙이는 것 가능

![[Pasted image 20250816191034.png]]

I2C 통신 버스는 SDA와 SCL 2개의 전선으로 구성

각 전선은 풀업 저항을 통해 전원에 연결, 각 장치의 단자 회로는 open-drain 혹은 open-collector 형식으로 구성, wired-AND 방식으로 동작

어느 하나가 낮은 전압 값을 출력하면, 다른 장치들의 출력 값에 상관 없이 낮은 값을 가짐

SDA는 통신을 위한 선, SCL은 클럭 신호를 위한 선

버스에 연결된 각 장치는 주소를 이용해 구분

I2C 통신은 하나의 버스에 연결된 여러 개의 장치들 사이에서 마스터-슬레이브 방식으로 통신 진행

통신의 시작 동작과 종료 동작은 마스터 장치에 의해 실시, 통신의 방향은 양 방향 모두 가능

즉, 마스터 장치가 슬레이브 장치로 데이터를 전송하거나, 슬레이브 장치가 마스터 장치로 데이터를 전송하는 것 가능

표준 통신 속도는 100Kbps, 최대 3.4Mbps까지도 가능

### I2C 통신의 비트 형식

- 동기 시리얼 통신 방식
- SCL의 클럭 신호와 SDA의 데이터 비트가 동기화
- 아래 그림은 I2C 비트 형식을 보임
- 클럭 신호가 높은 값인 구간 동안은 SDA의 데이터 비트 값은 변화 불가, 값을 유지
- SDA의 데이터 비트 값의 변경은 SCL이 낮은 값을 갖는 구간에서만 허용
![[Pasted image 20250816191041.png]]
### I2C START / STOP 상태

- I2C 통신은 START 상태로 시작, STOP 상태로 끝남
- 아래 그림은 START/STOP 상태를 보임
- 데이터 비트는 SCL 클럭 신호가 높은 값을 갖는 구간에 변경이 허용되지 않음
- START/STOP 상태에서는 클럭 신호가 높은 값을 갖는 구간에서 SDA 값이 변경
- START 상태는 클럭 신호가 높은 값을 갖는 구간에서 SDA값이 높은 값에서 낮은 값으로 변경
- STOP 상태는 클럭 신호가 높은 값을 갖는 구간에서 SDA 값이 낮은 값에서 높은 값으로 변경
- START/STOP 상태는 마스터 장치에 의해 생성
![[Pasted image 20250816191048.png]]
* 마스터 장치는 STOP 상태를 생성하기 전이라도, START 상태를 재생성 사능
* 슬레이브 장치를 변경하는 경우 혹은 데이터 전송 방향을 바꾸는 경우에 마스터 장치는 START 상태를 재생성

### I2C 데이터 형식
* 8개의 데이터 비트와 1개의 ACK 비트로 구성
* 데이터를 전송하는 장치는 8개의 데이터 비트를 전송, 수신 장치의 ACK 신호를 기다림
* 데이터 수신 장치는 8개의 데이터 비트를 받은 후, 클럭 신호가 높은 값을 갖는 구간 동안 SDA를 낮은 값으로 유지하는 ACK 신호로 응답
* 아래 그림은 I2C 데이터 전송 방식을 보임
![[Pasted image 20250816191305.png]]
* ACK로 응답하는 구간 동안에 데이터를 수신하는 장치가 SDA 값을 높은 값으로 유지하는 경우 존재 가능
* 이러한 경우 NACK이라 함, 데이터 수신 장치는 필요에 따라 NACK 신호로 응답하는 경우 존재

### I2C 주소 전송 형식
* 마스터 장치는 통신을 시작하기 위해 START 상태를 만든 후 슬레이브 장치의 주소를 전송
* 마스터 장치가 전송하는 주소는 7비트의 주소 값과 1비트의 R/W 비트로 구성
* 아래 그림은 I2C 주소 전송 형식을 보임
![[Pasted image 20250816191432.png]]
* 마스터 장치는 통신을 하려는 슬레이브 장치의 주소 7비트 뒤에 R/W 비트를 추가한 8개의 비트를 전송한 후 슬레이브 장치의 ACK 신호를 기다림
* 마스터 장치가 슬레이브 장치에 데이터를 쓸 경우 R/W=0 (write 동작)
* 마스터가 슬레이브 장치의 데이터를 읽는 경우 R/W=1 (read 동작)

### 마스터가 전송한 데이터를 슬레이브가 수신하는 경우
* 아래 그림은 마스터가 전송한 데이터를 슬레이브가 수신하는 경우의 통신 데이터 구성
* 마스터는 START 상태를 생성, 슬레이브 주소 전송
* 7비트의 슬레이브 주소 뒤 R/W 비트 추가
* 마스터가 쓰는 동작을 하게 되므로 R/W 비트는 0
* 이후 슬레이브 주소에 해당되는 슬레이브 장치는 ACK 비트로 응답(ACK=0)한다
* 마스터는 데이터 전송 후 STOP 상태 생성, 통신 종료
![[Pasted image 20250816191658.png]]
* 슬레이브 장치가 데이터를 수신 중에 데이터 수신을 중단 하려는 경우, 슬레이브 장치는 NACK으로 응답
* 해당 경우 마스터 장치는 STOP 상태를 생성해 통신을 종료하거나, START 상태를 재생성

### 슬레이브가 전송한 데이터를 마스터가 수신하는 경우
* 아래 그림은 슬레이브가 전송한 데이터를 마스터가 수신하는 경우의 통신 데이터 구성
* 마스터는 START 상태를 생성, 슬레이브 주소 전송
* 7비트 슬레이브 주소 뒤 R/W 비트 추가
* 마스터가 읽는 동작 -> R/W 비트는 1
* 슬레이브 주소에 해당되는 슬레이브 장치는 ACK로 응갑
* 마스터는 수신 상태로 전환, 데이터를 기다리고, 슬레이브 장치가 데이터를 전송
* 각 데이터 바이트에 대해 마스터가 ACK으로 응답
* 마스터는 모든 데이터가 수신된 후, STOP 상태를 생성, 통신 종료
![[Pasted image 20250816192124.png]]
* 마스터 장치가 데이터를 수신하는 중에 수신된 데이터 바이트에 대해 NACK으로 응답하면 슬레이브 장치는 전송을 중단
* 이후 마스터 장치는 STOP 상태를 생성, 통신 종료
