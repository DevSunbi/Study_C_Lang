## 템플릿
#### 함수 템플릿
* 범용 함수 만들기
	* 다양한 데이터 형식의 매개변수를 2개 입력 받아 덧셈하는 범용 함수
	* 데이터 형식별로 거의 동일한 코드를 반복해 만들어야 함
	* 처리할 수 있는 데이터 형식을 추가하려면 함수도 함께 추가해야 함
* 템플릿
	* 함수(또는 클래스) 하나로 다양한 형식의 데이터를 같은 알고리즘으로 처리
		* int add(int, int) -> complex add(c1, c2) 형식 => 오버로딩
			* 동일 형식, 다른 데이터
```cpp
template <typename T>
T data_sum(T operand1, T operand2){
	return operand1 + operand2;
}
```
* 함수 템플릿 정의, 함수 호출 시 선언한 데이터 타입에 따라 함수의 body가 상세 정의(자동 생성)
* 범용성 확보
	* 템플릿은 다양한 데이터 형식 처리
	* 모든 데이터 형식에 대응 가능한 알고리즘 정의 필요
	* 데이터 형식 고려 안하면 문제 발생
* 위 예제에서 데이터 타입 배열 할당 -> int* 형태로 선언되는데 이는 불가능
* 또한 배열의 크기가 다르면 T의 모호성 발생, 충돌
* 템플릿의 인스턴스화
	* 함수와 지역 변수는 컴파일시 스택 메모리 크기가 결정
		* 매개 변수에 맞는 데이터 형식에 맞는 함수의 메모리 크기가 정해져야 함
	* 함수 템플릿이 호출하는 구문 컴파일을 할 때 실행 가능한 코드를 만드는 작업
		* 템플릿 매개변수 데이터 형식 추론
		* 추론 형식 사용 오브젝트 코드 생성
* 템플릿은 데이터 형식 추론이 중요 단계
	* 인스턴스화 진행을 위해서 데이터 형식 추론 중요
	* 추론 불가 -> 인스턴스화 불가 -> 오류
		* 앞뒤 코드로 형식 추론이 어렵거나 암시적 형변환이 어려운 경우 오류
* 명시적 호출
	* 형식 추론 과정이 필요 없도록 명시적으로 데이터 형식 알려줌
	* 함수 템플릿 호출문에서 인자를 입력하기 전에 꺾쇠 괄호로 데이터 형식 명시
* 템플릿의 특수화
	* 특정 데이터 형식에 대한 예외적인 처리 방법 템플릿 특수화
	* 모든 템플릿 매개변수를 특정 데이터 형식으로 지정하는 명시적 특수화
		* 함수 템플릿은 명시적 특수화만 사용 가능
		* 특정 데이터 타입에 예외 설정 및 지정 코드 실행
	* 일부 템플릿 매개변수만 특정 데이터 형식으로 지정하는 부분 특수화
		* 부분 특수화는 함수 템플릿, 클래스 템플릿 모두에 사용 가능
		* 일부 지정, 나머지는 컴파일러가 추론
* default param
	* param에 기본 값을 설정 해 넣는 것

	* 함수 템플릿에서 사용된 템플릿을 클래스로 확장한 것
		* 멤버 변수 데이터 형식, 멤버 함수의 데이터 형식을 템플릿으로 변경

* 클래스 템플릿에서 형식 추론

		* cpp17 표준 이후부터는 클래스 템플릿에서도 형식 추론 가능
#### 중첩 클래스 템플릿
* 일반 클래스와 동일하게 중첩 클래스 사용
	* 중첩된 클래스 템플릿(안쪽 클래스), 기존 클래스 템플릿(바깥쪽 클래스)
	* 안쪽 클래스에서 바깥쪽 클래스 템플릿 매개변수를 사용하거나 새로 정의
	* 바깥쪽 클래스의 템플릿 매개변수를 사용하면 안쪽 클래스에서도 같은 데이터 형식으로 사용
* 활용법
	* 첫 번째 : 안쪽 클래스를 멤버 변수로 사용
	* 두 번째 : 독립된 클래스 객체로 사용
* 템플릿 함수는 바디가 없음(인스턴스화 안되어있음) -> 우선 순위는 일반 함수가 높음
#### 클래스 템플릿 프렌드
* 두 가지 규칙 준수
	* 인스턴스화 순서, 프렌드 규칙
* 전역에서 사용 가능한 프렌드 함수 선언 가능
* 가독성 향상을 위해 선언과 정의 분리
	* 선언과 정의 분리 -> 템플릿 매개변수 한 번 더 선언

## 파일 시스템
#### 파일 시스템 라이브러리
* 절대 경로와 상대 경로
	* 컴퓨터상의 모든 파일에는 해당 파일의 위치를 나타내는 고유 주소 -> 경로
	* 절대 경로 : 최상이 디렉터리인 루트에서 내가 원하는 파일까지의 전체 경로
	* 상대 경로 : 현재 위치를 기준으로 한 경로
*  path를 활용해 파일 경로 활용 가능
* ifstream은 파일을 읽기 위한 입력 스트림 제공
	* getline 함수로 파일에서 한 줄씩 읽어 line 저장 후 화면에 출력하는 동작 끝까지 반복
	* inFile.close 함수 호출, 파일 닫음, 파일 자원 해제 후 접근 종료
## 기타 유용한 함수
#### 난수 생성
* 난수 : 정의된 범위에서 무작위로 추출된 임의의 수
	* 난수는 그 다음에 나올 값 확신 불가, 시드라는 시작 숫자를 이용, 무작위성 증가
		* 시드 값으로 현재 시각을 사용하는 경우 많음
	* rand, srand 사용
		* rand는 난수 생성 패턴 하나, srand는 난수 생성 패턴을 여러 개
	* random 헤더 파일을 사용해 std::mt19937_64 사용을 통한 난수 생성 가능
* HW 엔트로피 사용 : 시스템에서 발생하는 무작위성의 정도
	* 마우스 움직임, 커서 위치, 키보드 입력, 디스크 I/O 등을 활용, 엔트로피 수집
	* random_device는 대체로 mt19937 엔진보다 느림
#### 수학 함수
* cmath 헤더 파일 이용, 삼각함수, 로그함수, 지수함수 등 수식 사용 가능

#### 복사 함수
* 얕은 복사 : 주솟값
* 깊은 복사 : 실제 값 새로운 메모리 공간에 복사
* copy함수로 깊은 복사가능
