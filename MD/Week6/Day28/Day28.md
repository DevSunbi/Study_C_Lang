#### Operating System
* User<->HW 사이 관리자의 형태로 존재
	* 사용자가 직접적으로 OS에 명령을 내릴 수 있게 함
		* 운영체제가 지원하는 Interface, 즉 내장 명령어를 알아야 함
	* 운영체제를 통한 체계화, 명령어의 형태 제공을 통한 제어(대화형)
	* 구조적으로 조합해 다양하고 복잡한 일을 할 수 있도록 지원
	* HW 제어 및 자원 컨트롤을 수행 -> OS는 HW에 더 가까움
* Application은 사용자와 더욱 가까움 -> 사용자가 복잡한 작업을 수행함에 도움

#### Kernel
![[Pasted image 20260429093020.png|320]]
* HW를 초기화해 사용 할 수 있게 하는 운영체제 핵심 부분
* 여러 SW가 OS에서 잘 작동할 수 있도록 메모리와 프로세스 관리, 네트워크 연결하는 등 주요 기능 제공
#### Device Driver
![[Pasted image 20260429093307.png]]
* HW 장치 초기화, OS와의 통신을 담당하는 SW
* 장치 : CPU, Memory, Mouse, HDD 등
#### File System
* file : 정보를 저장하는 기본 단위
* file system : HDD, SDD 등 저장 장치에 파일을 저장하고 관리하는 SW
* 파일 시스템은 파일의 이름, 크기, 저장 위치를 저장
* 파일에 대한 읽기/쓰기/실행을 제어
* 파일의 권한을 관리하고 저장 장치에 대한 접근도 제어
* 윈도우 : 분할
* 리눅스 : 하나
#### Network System
* Network Driver : 유선 랜, 와이파이, 블루투스 등 네트워크 장치를 초기화, 사용 가능하게 하는 장치 드라이버
* Network Stack : 네트워크 장치를 통해 들어온 네트워크 트래픽을 처리하는 네트워크 프로토콜을 구현한 SW
* Network Protocol : 네트워크에서 데이터를 주고 받는데 적용되는 통신 규약
#### 프로그램 실행과 멀티 태스킹
* CPU(Central processing unit) : 프로그램 실행에 필요한 기본 명령어를 실행하는 하드웨어
* 프로세스 : 실행중인 프로그램
![[Pasted image 20260429101509.png]]
* 멀티 태스킹
	* 기본적인 형태 : 시분할 기법 -> 특정 시간동안만 process 진행
#### process
* Process Scheduling : 프로세스 중 하나를 골라 CPU가 실행
* Scheduling Algorithm : 어떤 프로세스를 선택할지 결정하는 방법
* Process Scheduler : 스케줄링하는 주체
* Multi_Tasking : 운영체제가 프로세스 스케줄러를 사용, 여러 프로세스 동시 처리
#### Interrupt
* Interrupt : HW가 SW에 보내는 신호
#### Memory 관리
* 커널이 사용하는 메모리, 하드웨어에 할당된 메모리 등 사용 목적에 따라 구분
* 메모리 주소 체계 : 물리 메모리 주소, 가상 메모리 주소등으로 구분해 관리
* 가용 메모리가 부족하면 메모리 영역의 일부를 디스크로 잠시 옮겨놓고 메모리 사용
* 스와핑 : 디스크에 저장된 데이터를 메모리로 복구해 사용
#### User Interface
* User Interface : 사람이 컴퓨터 시스템을 사용하게 하는 방법
* Graphical User Interface : 그래픽을 통해 사용자와 상호작용
* Command Line Interface : 텍스트 기반 명령어를 통해 사용자와 상호작용

## 리눅스 
#### 리눅스 배포판
* 리눅스 배포판 : 운영체제의 목적에 맞게 리눅스 커널에 여러 프로그램을 묶어 제공하는 패키지
* 종류
	1. 레드햇 리눅스 : 대표적인 유료 배포판
	2. 센토스 : RHEL 소스코드 기반 무료 배포판
	3. 우분투 : 대표적인 무료 배포판
* 사용
	* 서버 : IT 서비스 구성하기 위한 서버
	* 클라우드 컴퓨팅 : 클라우드 서비스 구축을 위한 백엔드
	* 임베디드 시스템 : PC나 서버 환경에 비해 제한적인 자원을 가진 경우
	* 모바일 기기 : 스마트폰이나 태블릿 등
## 터미널과 쉘
#### 터미널
* 터미널 : 컴퓨터와 사용자 간에 상호작용 할 수 있게 연결하는 장치
* 입력장치 : 사용자가 컴퓨터에 명령을 전달하는 장치
* 출력 장치 : 컴퓨터가 사용자에게 결과를 보여주는 장치
#### 셸

* Bash : 리누스 토르발즈가 리눅스를 개발할 때 처음 포팅한 프로그램
