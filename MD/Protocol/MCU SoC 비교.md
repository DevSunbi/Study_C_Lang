### AP(SoC) vs MCU : 시스템 설계 차이
1. [[MCU]]
	* 정의 : CPU, 메모리, 주변 장치가 하나의 칩에 들어있는 저전력 제어 장치
	* 역할 : 브레이크, 에어백, 윈도우 제어 등 단순하지만 즉각적이고 안정적인 제어
	* 핵심 : 자원이 제한적인 환경에서 신뢰성 높은 제어 수행

2. AP(Application Processor / [[SoC]]) 
	* 정의 : MCU보다 강력한 연산 능력을 갖춘 고성능 칩, 보통 OS 위에서 작동
	* 역할 : 자율주행 경로 계산, AI 영상 분석 등 복잡한 데이터 처리 담당
	* 핵심 : 대용량 데이터를 고속으로 처리하기 위한 멀티코어 기반 통합 시스템 칩

### SoC와 MCU의 기술적 차이점 3가지
1. 메모리 구조(Internal vs External)
	* MCU : 메모리(RAM, Flash)가 칩 안에 내장(On-chip Memory)
		* 칩 하나로 모든게 해결, 보드 사이즈를 줄일 수 있고, 데이터 전송 경로가 짧아 실시간 제어에 유리
	* SoC : 연산 성능 강력, 대용량 메모리 필요
		* 보통 칩 외부에 별도의 DDR 메모리 연결, 이를 관리하기 위한 메모리 컨트롤러 내장

2. 연산의 성격 (Deterministic vs Throughput)
	* MCU(Determinism) : "정해진 시간 안에 결과를 내놓는가"가 중요(에어백)
	* SoC(Throughput) : "한 번에 얼마나 많은 데이터를 처리하는가"가 중요(영상데이터)

3. 부팅 프로세스
	* MCU : 전원을 켜자마자 코드의 첫 줄부터 실행, 부팅 속도가 거의 즉시에 가까움
		* HW의 제약으로 인한 Non-OS, RTOS 사용
	* SoC : HW 초기화(Init)하고 OS 커널을 메모리에 올리는 부팅 과정이 필요, 수 - 수십초 소요
