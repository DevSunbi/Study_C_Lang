## 상속성과 다형성
#### 상속성
* 클래스를 만들 때 다른 클래스의 멤버 함수, 멤버 변수를 물려받는 것
* 이미 추상화, 캡슐화된 클래스를 확장, 변형하는 방법
	* 확장 : 기존 물려 받고, 추가(속성, 함수)
	* 변형 : 기존 물려 받고, 수정(상속 받은 것을 본인만의 특징으로 변경)

| 상속 해주는 클래스 | 상속 받은 클래스 |
| ---------- | --------- |
| 부모 클래스     | 자식 클래스    |
| 기본 클래스     | 파생 클래스    |
| 슈퍼 클래스     | 하위 클래스    |
* 공격 특성이 다른 3 몬스터를 각각 정의하면 특수 공격 외 코드 중복
* 3 몬스터의 공통 특성과 행동을 가진 부모 클래스를 설계 후 상속 -> 중복 최소화
	* 3 몬스터가 모두 동일한 특성을 가졌다는 논리적 관계를 설계에서도 확인 가능
* 메모리 상, 부모 객체 선언 시 stack에 부모 객체의 공간 할당, 변수 또한 할당
* 자식 객체 생성 시, stack에 공간 할당, 메모리에 객체 공간은 존재, 부모 객체에서 private로 생성되어있으면 child에서 조차 접근 불가한 private의 특징
* 클래스 치환
	* 상속의 중요한 여러 특성 중 하나, 다형성의 기반
	* 자식 클래스는 부모 클래스 대체 가능
* 자식 클래스가 부모 클래스의 속성과 기능을 변경하면 안됨
	* 부모 클래스가 public으로 선언한 함수의 시그니처를 변경 없이 상속 받아야 함
	* 부모 클래스의 모든 제약과 동작 방식 그대로 유지
* 자식 클래스는 언제든 부모 클래스를 대체할 수 있어야 함
	* 리스코프 치환 원칙
* 단일 상속
	* 하나의 부모로부터 상속
* 다중 상속
	* 상속 받을 부모 클래스 나열
	* 상속 접근 지정자 명시 안하면 private으로 지정
	* 이름 충돌 및 모호성 발생 가능
		* A class의 i와 B class의 i를 상속받은 C에서 모호성 발생, A::i, B::i 선언 필수(더러움)
	* 다이아몬드 문제
		* A를 상속받는 B, C / B, C를 상속받는 D 이 때 의도치 않은 충돌 발생 가능
		* 함수 오버라이딩 시 어느 함수를 실행해야 할지 모호함이 발생(지정해야함)
#### 다형성
* 상속 받은 자식 클래스들이 부모와 다른 다양한 특성을 가지는 것
	* 동일한 함수가 부모와 다른 행동 방식으로 동작하는 것
	* 상위 개념의 역할을 대신할 수 있으면서 각자의 고유한 특징으로 동작하는 것
* 하위 호환성을 처리할 때도 다형성이 사용
	* 파일 포맷이 변경되면서 파일을 처리하는 프로그램 버전이 변경되어도 예전 파일 포맷을 처리하는 것
* 객체지향에서 다형성을 구현하려면 함수의 오버라이딩을 사용
* 상속받은 함수는 부모 클래스에서 정의된 내용 그대로 사용
	* 다형성을 구현하려면 함수의 시그니처는 그대로 유지, 다르게 정의해야 함
* 시그니처는 그대로 유지하면서 부모 클래스에 정의된 함수를 재정의하는 것
	* 오버라이딩
* 업캐스팅 후 오버라이딩 된 함수 호출
	* 자식 클래스의 객체를 부모 클래스 타입의 포인터나 참조로 가리키는 것
	* 동적 바인딩 된 상태라면 부모 클래스 타입으로 가리켜도 virtual로 인해 자식의 함수 실행
	* 가상 함수를 활용한 함수 오버라이딩
		* 가상 함수를 활용
			* 멤버 함수의 이름 앞에 virtual이라는 키워드로 선언하면 가상 함수
			* 업캐스팅을 해도 자식 클래스에서 오버라이딩한 함수 호출
#### static/dynamic binding
* static binding(정적 바인딩, Early binding) -> default
	* 컴파일 타임 바인딩 => 컴파일 타임에 결정
	* 객체가 아닌 type을 보고 결정
* dynamic binding(동적 바인딩, runtime binding / late binding)
	* keyword : virtual 
	* 실행 시 결정 => 객체의 타입 확인 후 그에 맞는 함수 찾아 실행
	* virtual func table에 저장되어있는 함수의 주소(객체의 vptr 확인)로 jump

#### 생성자와 소멸자
* 생성자 : 객체가 생성된 직후에 자동으로 호출되는 함수
	* 상속 받은 자식 클래스의 생성자가 가장 나중에 호출
	* 부모 클래스의 생성자가 먼저 호출
		* 다중 상속일 경우 상속받은 순서에 따라 차례로 호출
	* 자식 클래스에서 생성자가 없어도 부모 클래스 참조 후 인자가 없는 생성자를 컴파일러가 만들어줌(Default Construct)
	* 오버로딩 시 인자 리스트 달라야함(리턴 타입, 이름은 동일해야함)
	* 객체 생성 시 특정 값을 생성자로 전달 가능
		* 생성자에 매개 변수를 추가, 생성자에 전달된 매개변수에 맞는 생성자가 호출
	* 생성자에서 다른 생성자 호출
		* 생성자에서 다른 생성자를 호출해야 가능
* 멤버 변수 초기화
	* 클래스는 멤버 변수 여러 개 가질 수 있음
		* 멤버 변수는 객체 생성 후 동작 중에 값 정해짐
		* 일부 변수는 클래스가 생성될 때 값을 정해짐, 문법적으로 반드시 초기화해야하는 멤버 변수도 존재
	* 객체 생성 시 초기화 필요한 멤버 변수는 생성자를 이용
		* 레퍼런스 멤버 변수와 상수 멤버 변수는 생성자에서 반드시 초기화, 정적 멤버 변수는 전역 범위에서 초기화
	* 일반 멤버 변수 초기화
		* 대입 연산으로 초기화, 생성자 선언 뒤 쪽에 초기화 목록을 이용하는 방법
			* 초기화 목록을 이용하는 경우 대입이 아닌 값을 직접 초기화 구문 이용
			* 대입 연산은 데이터 복사, 복사 초기화
			* 복사 초기화는 Rvalue가 필요, 메모리 좀 더 사용
* 정적 멤버 변수 초기화
	* 함수의 정적 변수 함수 내에서 선언하고 초기화 진행
	* 정적 멤버 변수는 선언된 모든 객체에서 동일하게 사용
		* 클래스 객체가 선언되기 전에 초기화 진행
		* 함수는 프로그램이 실행되면 메모리가 할당되지만, 클래스 멤버는 선언 후에 할당되기 때문에 차이 발생
		* 프로그램 실행 시점에 값이 지정되어야 함
			* 클래스에 선언한 정적 멤버 변수는 전역 범위에서 초기화 해야함
			* 전역 범위에서 초기화, 어떤 클래스에 속한 멤버인지 알 수 있도록 네임 스페이스 추가
