## 아스키 코드 값과 문자 입출력 함수
#### getchar 함수와 putchar 함수
* 변수명=getchar();
* putchar(변수명);
* getchar가 왜 int type?
	* 입력 되는 것은 문자 하나 -> 실제로는 아스키 문자 값에 해당되는 값이 전달
	* 개행 문자까지 IO Buffer에 전달 

#### 버퍼를 사용하는 입력 함수
* scanf 함수가 문자를 입력하는 과정
	* 키보드를 통해 입력 -> buffer에 전달, 프로그램 출력
	* Ctrl + z / -1 / \n => EOF임
	* scanf는 정상 리턴 시 1 반환, EOF 입력 시 -1 반환
	* %d = 10 => new line character
	* scanf는 숫자만 가져가고 \n을 버퍼에 남겨둠
		* 뒤 따라오는 getchar가 남겨진 \n을 본인 입력으로 착각
		* 입력 명령 중간에 getchar() 삽입, 반환값 저장 없이 버퍼 맨 앞 \n 소모
		* 또는 형식 지정자 앞에 공백 삽입을 통한 화이트 스페이스 무시 가능

## 문자열
#### 문자열 상수 구현 방법
* 문자의 배열, Null(\0)로 끝남 => 상수 테이블에 저장된 형태
* %p -> 시작 주소(포인터)를 찍음
#### char 포인터로 문자열 사용
* char 포인터 변수 선언, 상수 테이블에 문자열 저장
* 문자열(char 배열, 이름은 주소 상수) - 포인터에 저장 => 호환
* 선언된 변수의 주소를 쫓아가면 char을 만남
* 포인터와 배열은 동격, 배열은 주소 상수고 포인터는 주소 변수
	* 포인터 문자열 선언 시 문자열은 RODATA(읽기 전용 공간)에 저장되지만, 포인터 시작 주소 자체는 Stack에 저장된다(배열의 경우 데이터 자체도 Stack에 저장)
* %s로 scanf가 인자를 받을 시 white space 만나면 white space 이후 데이터는 무시(입력 버퍼에 남겨둠)
#### gets 함수를 사용한 문자열 입력
* 덩어리 단위 입력 받음 -> 공백과 같은 데이터도 유효 데이터 취급
* \n 이전 모든 데이터 읽은 후 \n을 \0로 변환처리
* 인자로 주소가 들어감
#### fgets 함수를 사용한 문자열 입력
* gets -> standard
* fgets -> file
* 반드시 file을 통해 데이터가 들어옴을 알 수 있다
* 현재 교과정은 file input 할 줄 모름 -> stdin을 통해 표준 입력장치 사용
* \n까지 데이터로 생각, \n 뒤에 \0이 붙음
#### 문자열을 출력하는 puts, fputs 함수
* Null 문자 만날 때 까지 출력
#### 문자열을 대입하는 strcpy 함수
* strcpy(p1, p2) -> p1에 p2 내용 복사
* 사이즈 고려하며 시행 필수
* 복사 후 문자열의 끝에 null 문자 자동 삽입
* 첫 인자가 배열 -> 실 공간 할당 -> copy에 문제 없음
* 첫 인자가 포인터 변수 -> 데이터 공간이 변수 자체가 가진 것이 아님
	* strcpy의 첫 인자에 포인터가 위치 할 수 없음
	* 즉, 가리키는 실 공간이 없으면 안됨
#### 문자열을 붙이는 strcat, strncat 함수
* 기존 문자열 뒤에 append 시행
	* 기존 문자열의 \0 위치를 찾아 해당 위치부터 복사 시작
* strncat(인자, 문자열, 갯수);
* 유효한 공간에 데이터가 저장되어있어야만 함
	* 복사 받는 곳은 배열이어야 함
* 문자열 최초로 붙일 때는 초기화 필수
#### 문자열 길이를 계산하는 strlen 함수
* 배열의 길이와 무관, 문자열의 길이를 계산함
* 유효한 공간에 데이터가 저장되어있어야만 함
* 시작 주소부터 null 문자 만날 때까지 갯수를 셈
#### 문자열을 비교하는 strcmp, strncmp 함수
* 갯수 지정시 strncmp 사용
* 제일 첫 글자부터 차례대로 비교
* ascii 코드 값으로 비교함
	* 대소문자 비교는? -> 소문자의 값이 큼

## 변수의 영역과 데이터 공유
* global / static 공간으로 나눠 생각 시작
#### 지역 변수
* 함수의 내부에 선언 되어있는 변수
* 선언된 block이 끝나면 저장 공간이 메모리에서 사라짐
* stack 공간이 끝나도 중첩된 block이 있는 경우 block에 따라 편이 존재
* 해당 block에 선언된 같은 이름의 변수가 없으면 외부 block에 존재 여부 확인
	* 해당 block에 있는 변수가 우선 순위를 지님
	* 외부 동일명의 변수는 local value에 의해 가려짐
* Global 변수에도 없으면 컴파일 에러 발생
* block 안쪽에 선언된 변수는 별도의 block
* 전역변수 -> block의 외부 존재
	* 어느 함수에서 값을 수정했는지 알 수 없음
* stack(automatic by scope rule) -> 초기화 하지 않고 재사용하는 공간(garbage 존재)
* heap(static by user) -> memory allocation func -> 포인터 변수에 연결해 주로 사용
#### 정적 지역 변수
* 변수 선언마다 메모리 할당 -> 함수 종료와 동시에 stack 공간 할당 해제
* static 선언 -> data segment에 메모리 할당 static 변수 할당한 함수에 해당 메모리 접근 권한 생김, 해당 공간은 자동 초기화 됨
* 값은 유지하며 해당 함수를 위한 전용 공간이 필요할 때 static 변수 사용
	* lifecycle이 local 변수처럼 사라지지 않고 계속 유지
	* 접근 권한은 선언부 함수에 종속
	* 타 함수는 static 변수 공간에 접속 불가
#### 레지스터 변수
* keyword register를 사용함
* 레지스터에 변수 생성, 메인 메모리를 사용하지 않고 연산 수행
* 그럼 번지수는? -> 없음
#### 주소를 반환하는 함수
* call by address
* static 공간인 경우 반환을 제대로 못하는 경우 존재
	* 정적 지역 변수나 전역 변수와 같이 함수가 반환된 후에도 저장 공간이 유지되는 경우에만 주소 반환
