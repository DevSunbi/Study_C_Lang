## 포인터
#### 메모리의 주소
* 주소는 변수가 할당된 메모리의 시작 주소 => 할당 시 매번 메모리 주소 할당(virtual memory address) -> 논리 메모리 공간은 실 메모리 주소와 mapping되어 사용
	* 메모리 공간은 프로그램 실행 시 프로그램이 새롭게 메모리를 OS로부터 할당
* 변수의 공간이나 값은 이름으로 사용
	* 변수 table -> 컴파일러가 먼저 만들어냄 => 변수의 타입 및 이름 명시, 메모리는 아직 할당 받기 전
	* 프로그램 실행 시 변수 테이블에 주소 할당
	* 가상 메모리 주소를 역참조, mapping되어있는 실 주소 접근, 변수 값 변경
* 주소를 알면 주소로도 변수의 공간이나 값 사용 가능
* 변수 : 값 저장 / 포인터 : 값이 들어가 있는 위치 저장(가상 메모리 주소)
#### 포인터 변수
* 타입 자체가 Datatype *
* 변수의 명은 \*을 빼고 얘기하는게 맞음
* 포인터 변수 : 해당 변수에 저장된 값은 주소 값
	* 값을 역참조, 역참조한 값의 주소로 이동해 해당 메모리 주소의 값을 읽어옴
	* 포인터 변수는 데이터의 타입과 무관하게 항상 8바이트로 저장(long long size)
		* Size 관점에선 맞으나, 의미와 동작 관점에서는 틀리다
		* long long의 경우 +1을 하면 값이 1 증가
		* 포인터의 경우 타입의 크기 만큼 주소 점프
		* 타입으로 쓰고 연산하고 싶으면 uintptr_t(intptr_t) 사용
* 초기화 시 Null로 초기화
* 메모리 주소 직접 할당은 가능하나, 허용된 영역인지는 미지수
	* 가상 메모리 영역은 할당마다 다른 주소를 부여 -> 가변 주소기에 어찌 될지 모름
#### const를 사용한 포인터
* const를 사용한 포인터는 간접 참조하여 값을 바꿀 수 없음
* 데이터 혹은 주소 자체를 고정함을 의미
	* 함수 인자 보호 혹은 HW 레지스터 고정을 위해 사용함
* const datatype* const val 이것도 가능함
	* int * const p : p is a const pointer to int
	* const int * p : p is a pointer const int
	* const int * const p : p is a const pointer to const int
* call by pointer adderess -> 함수에 변수의 값이 아닌 주소를 전달하는 방식
	* 함수 밖의 변수 값을 직접 바꾸고 싶을 때 필수
	* 효율성이 높음
	* 여러 개의 값 반환 가능
#### 주소와 포인터의 차이
* 주소는 상수, 포인터는 변수라 포인터의 값은 가변적
* 두 개 이상의 포인터가 하나의 변수 가리킬 수 있다
* 주소와 포인터의 크기 => 동일하다 생각(8byte)
* 포인터 변수에는 주소가 들어감 -> 잘못 들어가면 해석이 달라짐
	* 형 변환을 사용한 포인터의 대입은 언제나 가능
### pointer func
* 함수 내에서 포인터 사용 유무의 차이 기재
* 프로그램 실행
	* main 함수가 stack 공간 할당
		* swap func 만남
			* parameter mapping 시작
			* value copy
			* temp 초기화, x값 저장
			* x에 y 값 저장
			* y에 temp 값 저장
			* swap 함수 종료
		* swap 함수 종료 시 stack 공간 할당 해제
		* pointer 함수 만남
			* parameter mapping 시작 -> 포인터 주소 할당 및 저장
			* 전달 받은 주소를 통해 메인 함수의 가상 메모리 영역 진입, 값 수정
			* pointer swap 함수 종료
		* 원본 변수의 수정 사항 반영(mapping 후 가상 메모리 공간 진입, 수정 완료)

## 배열과 포인터
#### 배열 명으로 배열 요소 사용하기
* str은 이미 pointer로서의 시작 주소 보유
* 주소 연산 한해 a + 1 \* sizeof(datatype)
* address 기반 연산을 진행, 배열 요소 직접 접근 가능
	* int a[3];
	* int* pt = a;
	* => scanf("%d", a+1);
	* a+1 : 주소(포인터)
	* \*(a+1) : 그 주소에 들어있는 실제 값(역참조)
	* 따라서 a[1] == \*(a+1)
	* 연산은 가능하나 상수임, a++과 같은 연산자 사용 불가
	* int* pt = a; 인 경우 pt++가능 => pt는 변수임
		* 이때 pt++는 배열의 다음 요소의 주소와 동일함
* 포인터의 뺄셈 결과는 배열 요소 간의 간격 차이를 의미함
#### 배열 값을 출력하는 함수
* 함수로 배열 원본을 넘기지 않는 이유 -> 배열이 크면 함수 실행 시 스택에 메모리 할당 필요 => 메모리 손해가 극심하기에 배열 전체를 넘기지 않음
* 또한 직접적으로 메모리 접근 못하도록 막는 보안상의 이유
* 배열명 -> 하나의 포인터 변수 취급, 안전한 연산을 위해 함수로 넘길 때 size까지 넘김
