## 사용자 정의 자료형
#### 구조체
* struct(예약어) name {member}; => 이름이 곧 type
* typedef struct {}name; 형태도 가능
* 구조체변수명.멤버명 = val;
* 여러 개의 이중 데이터를 하나의 이름으로 묶어 하나의 데이터 타입으로 보기 위한 목적
* 멤버 별로 따로따로 메모리 공간을 갖게 된다
* 실행 효율을 위해 패딩 바이트를 넣어 바이트 정렬을 시행
	* sizeof로 구조체 사이즈 확인 -> (int, double)경우 16바이트 찍힘
	* 크기의 단위를 맞춘다 생각하면 좋음
	* 멤버 변수의 주소에 진입해 값 확인
	* address 계산 시 정규화된 형식으로 시행, 편리성을 위한 패딩 바이트
	* 가장 큰 자료형 기준으로 padding byte 크기 결정
	* 순서 배치 신경쓰면 메모리 낭비 줄일 수 있음
	* 진짜 메모리 극한으로 아껴야하면 예약어를 사용해 아낄 수 있음
		* 단, CPU의 접근 속도 저하, Alignment fault 가능성 존재
	* byte마저 아까우면 bit field 사용해 비트 단위로 쪼개 사용
#### 구조체 변수를 함수의 매개변수에 사용
* 매개변수와 반환형 모두 구조체로 선언
* call by value 형태
* 자료형이 같은 구조체 변수는 대입 연산 가능
#### 구조체 포인터와 -> 연산자
* 구조체도 포인터 사용 변수 생성 가능
* dot연산자의 우선순위가 높음 => 제대로 해석 안되니 (\*ps).val 형태로 작성
	* 혹은 ps->val 형태로 작성
* 구조체를 배열로 만들 수 있음 -> 동일한 형태가 여러 개 나옴
	* basetype 알아야함 -> struct 구조체
* dot은 직접 접근, ->는 구조체 포인터를 통해 간접 접근
#### 자기 참조 구조체
* 본인의 형태를 지닌 포인터 변수 존재
	* 구조체가 아닌 일반 변수가 존재 -> 재귀 함수와 동일, 탈출 조건 부재로 오류
#### 공용체
* 메모리 공유 형태
* struct처럼 예약어 union을 사용
* 하나의 메모리 공간을 여러 멤버가 사용(공유)
#### 열거형
* enum 키워드 사용
* 문자의 형태와 같이 데이터를 상수화
* datatype은 정의도니 타입을 명시
* 실제로는 숫자 0, 1, 2, 3과 같이 기재해도 똑같이 작동
* 관련 있는 상수들을 하나로 묶어 타입으로 관리 가능
	* #define 사용하면 되긴 하는데, 하나하나 해야해서 귀찮음
* 열거형의 메모리 크기는 int로 처리
* switch-case, register mapping 등 사용시 편리
* typedef와 함께 사용해 깔끔하게 사용 가능

## 파일 개방과 입출력
#### 파일 개방과 폐쇄
* DB와 연동하지 않아도 간단하게 읽고 쓰기를 지원하기 위해 파일 작업 필수
* file -> 구조체
* 파일 구조체의 포인터 변수를 이용, 파일 구조체를 접근해서 작업
* 작업 후 닫아줘야함 -> fopen - fclose
* read, write, append(읽기, 쓰기, 덧붙히기) 세 가지 형태, 쓰기는 상호 배타적
* fgetc -> 문자 입력 함수
* 버퍼의 내용을 모두 읽으면 EOF(-1)을 반환
* fputc -> 문자 출력 함수
* 버퍼 사용 안하면 진짜 한글자씩 출력 -> 수많은 작업 발생
#### + 개방 모드, fseek, rewind, feof 함수
* r+ : 텍스트 파일에 읽고 쓰기 위해 개방
* w+ : 텍스트 파일의 내용을 지우고 읽거나 쓰기 위해 개방
* a+ : 텍스트 파일을 읽거나 파일의 끝에 추가하기 위해 개방

#### fscanf, fprintf
* 파일 포인터 앞에 주면 printf, scanf 가능
##### fflush 함수
* 강제로 내보내는 함수
* 스트림 버퍼의 내용을 즉시 장치로 기록
#### fread, fwrite
* 소켓 이용 통신에서 byte 단위로 통신(파일에 읽고 씀)
* 데이터 크기 지정, 입출력 가능

## 전처리 지시자
#### 파일 포함 include
#### 매크로명을 만드는 define
* 매크로 함수 사용 가능
* 근데 매크로 함수는 함수가 아님
* 문자열의 대치로 됨 -> 연산자의 우선순위와 섞은 경우 오류 가능성 존재
#### if, endif, else, ifdef 조건부 컴파일
* 버전에 맞춰 조건부 컴파일 할 때 사용
* ifdef -> 정의 되어있으면 T, 없으면 F => 참일 시 아래 명령문 실행
#### pragma 지시자
* padding을 없애는데 pack 사용
#### 분할 컴파일 방법
* 함수 파일 분리한 후 목적파일 병합을 통해 시행
	* 헤더파일 사용
* extern/static
	* extern -> 타 파일 선언
	* static -> 해당 파일 내에서만 존재

# C++ 시작
#### 입출력 비교
* scanf, printf 둘 다 가능
* iostream에 정의 되어있는 cin, cout 사용 가능
#### namespace
* 변수나 함수가 속해 있는 소속을 지정해 주는 역할
* 내부 식별자(형식, 함수, 변수 등) 범위 부여, 라이브러리간 이름 충돌 방지
* std -> std라는 네임 스페이스에 접근하는 표현
* using namespace를 사용하면 std:: 생략 가능
	* 생략된 네임스페이스로 범위 착각 가능함
		* 혼선 야기 가능성 존재, 고려해서 사용
#### cout 출력
* 출력 자료형 지정 없이 정수, 부동 소수점등 출력 가능
* << 연산자를 사용, 출력 대상을 cout 스트림 객체로 전달, 여러 개 변수 전달 가능
* 개행 문자는 endl을 cout 스트림 객체에 전달
#### cin 입력
* 입력 자료형 지정 없이 정수, 부동 소수점 등을 입력 가능
* >> 연산자를 사용, 출력 대상을 cout 스트림 객체로 전달, 여러 개 변수 전달 가능
#### wchar_t
* 1byte로 표현이 불가한 언어를 와이드 문자라 지칭
* 2byte로 정의 되어있으나, GCC 컴파일러 기준 4byte
