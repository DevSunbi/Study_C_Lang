## STL의 컨테이너와 알고리즘
#### 컨테이너?
* 컨테이너 : 같은 타입의 여러 객체를 저장 가능한 묶음 단위의 데이터 구조
	* 데이터 저장, 관리, 저장된 원소에 접근할 수 있는 멤버 함수 제공
	* 한 가지 타입의 객체들만 보관 가능, 탐색/저장 알고리즘은 라이브러리가 제공
* 반복자(이터레이터) : 컨테이너의 내부 원소들을 순회하는 객체
	* 표준 템플릿 라이브러리에서 모든 컨테이너는 각자의 반복자를 제공
#### Vector
* 가변 길이의 배열처럼 선언
* push_back으로 원소 추가
* begin 함수는 벡터의 첫 번째 우너소에 접근할 수 있도록 위치 반환
* 반복자 it에 포인터처럼 역참조 연산자 이용, 해당 원소에 접근 가능
#### 반복자
* for (vector \<int>::iterator it = vec.begin(); it != vec.end(); it++){ }
* 컨테이너 순회
	* vec begin 함수를 호출, vec의 시작 위치로 초기화
	* vec 컨테이너를 한 칸씩 순회하면서 원소 출력(\*it)
	* end 함수는 컨테이너 저장된 맨 마지막 원소의 바로 다음 위치 반환
	* 빈 상태라면 begin이랑 end는 같은 위치
#### 순차 컨테이너
* 컨테이너는 종류에 따라 다음 3가지 범주
	* 순차 컨테이너 : 데이터가 순서대로 삽입되는 컨테이너
	* 연관 컨테이너 : 데이터가 오름차순/내림차순처럼 미리 정의된 순서로 삽입되는 컨테이너, 연관 컨테이너 안에 있는 원소들은 정렬된 상태 유지
	* 컨테이너 어댑터 : 특별한 자료구조를 표현한 컨테이너
* 순차 컨테이너
	* 원소들이 선형으로 배열된 데이터 구조, 각 원소는 위치에 따라 접근
	* 배열
		* 고정된 크기의 배열을 담는 컨테이너
		* 일반 배열 변수 -> 포인터로 변환 시 배열 길이 정보 알 수 없음
		* 동적 배열 메모리 할당/해제 직접
		* array 컨테이너 초기화는 일반 배열 동일
			* 5개 배열을 갖는 컨테이너 선언, 더 많은 값 넣으면 컴파일 오류
		* 일반 배열처럼 \[] 연산자로 각 원소에 접근
		* 배열의 유효 범위 밖의 인덱스로 접근을 시도, 런타임 오류 발생
		* at 함수에 배열의 인덱스를 전달하면 해당 원소에 안전하게 접근
			* 범위 벗어나면 at 함수가 out_of_range 예외 발생
			* at 함수는 유효 범위를 검사, \[]연산자보다 느리지만 안전한 코드 작성 가능
		* size 함수로 배열의 크기 알 수 있음
			* 메모리에서 차지하는 크기가 아닌 원소를 저장할 수 있는 개수
	* 벡터 : 동적인 길이의 배열 컨테이너, 배열의 길이가 가변적
	* 벡터의 크기는 힙 메모리가 허용하는 한 계속 증가
	* 값을 변경 못하는 정적 반복자 -> const_iterator 반복자가 가리키는 원소값 변경 불가
	* ![[Pasted image 20260409111643.png]]
	* -> vector const 사용법
* ![[Pasted image 20260409111848.png]]
* 범위형이 좀 더 직관적
* const_iterator 주소 변경은 가능, 내용만 못바꿈
* reverse_iterator : 역동작 반복자
* shrink_to_fit -> 메모리 함수를 fit하게 맞추는 개념, 현 데이터 사이즈에 맞게 공간 축약
* 연결 리스트 데이터 구조 구현 컨테이너
	* 연결 리스트는 데이터가 노드라 불리는 개별 객체로 구성
	* 각 노드는 다음 노드를 가리키는 포인터 포함
	* 데이터 삽입, 삭제하는데 효율적, 순서 유지
	* push_back, push_front, erase, insert 
	* empty, size
* dequeue : 벡터 구조 구현 컨테이너
	* 여러 개의 메모리 블록을 나눠 배열 저장, 저장 공간 부족 시 새로운 블록 만들어 연결
	* 벡터는 메모리 부족하면 새로 할당, 원솟값 모두 복사
#### 연관 컨테이너
* 데이터를 저장하고 검색할 때 사용 -> 특정 키를 사용해 효율적인 검색 지원
* 내부 구조는 트리나 해시 테이블과 같은 형태로 검색에 최적화된 구조
* set : 고유한 값을 정렬된 상태로 저장
	* 이진 탐색 트리를 사용해 원소 저장
		* 삽입, 삭제, 검색 시간 복잡도가 nlong 근접
		* 정렬 유지 정렬 컨테이너
* map : 키-값 쌍을 이룸
* 멀티맵 : 중복 값을 허용하는 멀티 맵 특징
	* 중복된 키 저장 : 중복 키 허용, 같은 키에 여러 값 저장 가능
	* 자동 정렬 : 키 기준 정렬, 기준 정의 가능
	* 이진 탐색 트리 : 이진 탐색 트리 기반
#### 컨테이너 어댑터
* 컨테이너 어댑터
	* 다른 컨테이너를 기반으로 새로운 기능 제공
	* stack 어댑터 -> vector 기반, LIFO 
	* queue 어댑터 -> deque 기반, FIFO
