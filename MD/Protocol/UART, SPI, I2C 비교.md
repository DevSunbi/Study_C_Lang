| 구분          | **UART**                              | **SPI**                                     | **I²C**                                |
| ----------- | ------------------------------------- | ------------------------------------------- | -------------------------------------- |
| **배선**      | TX, RX (2선, GND 제외)                   | MOSI, MISO, SCLK, CS (4선+장치당 CS 추가, GND 제외) | SCL, SDA (2선, GND 제외)                  |
| **통신 방식**   | 비동기 (Start/Stop bit, Baud rate 맞춤 필요) | 동기 (클록 공유, 풀 듀플렉스)                          | 동기 (클록 공유, 반이중, 멀티마스터 지원)              |
| **속도**      | 수백 kbps ~ 수 Mbps (보통 1~2 Mbps 내외)     | 수 MHz ~ 수십 MHz (10~50 MHz 이상도 가능)           | 표준 100 kbps, 패스트 400 kbps, 고속 3.4 Mbps |
| **장치 연결**   | 1:1 전용 통신                             | 1:N (CS 핀 개수만큼 확장)                          | 1:N (주소 기반, 최대 127개 슬레이브)              |
| **데이터 전송**  | 반이중(Full-duplex 아님), 단순               | 풀 듀플렉스                                      | 반이중                                    |
| **하드웨어 요구** | 단순, 거의 모든 MCU 내장                      | 클록·라인 많음, 회로 복잡                             | 풀업 저항 필요, 버스 공유                        |
| **에러 검출**   | 패리티 비트 가능, 비교적 단순                     | 기본적으론 없음(상위 프로토콜 필요)                        | ACK/NACK 신호 제공                         |
| **주요 사용처**  | PC ↔ MCU, 디버깅, GPS, 블루투스 모듈           | 센서(고속), 디스플레이, 플래시 메모리                      | 저속 센서, RTC, EEPROM, 주변기기 제어            |
# 📌 장단점 정리

### 1. [[UART(Universal Asynchronous Receiver Transmitter)]]

- ✅ **장점**
    
    - 배선 간단 (2선)
        
    - 구현 쉬움, MCU 대부분 내장
        
    - 디버깅/로그 출력에 최적
        
- ❌ **단점**
    
    - 1:1 통신만 가능 (멀티드롭 불가)
        
    - 클록 동기화가 없어서 Baud rate 맞추어야 함
        
    - 속도·신뢰성 한계 (보통 1~2 Mbps 수준)
        

---

### 2. [[SPI(Serial Peripheral Interface)]]

- ✅ **장점**
    
    - **매우 빠름** (수 MHz~수십 MHz)
        
    - **풀 듀플렉스** 전송 가능
        
    - 프로토콜 단순 → 구현 쉬움
        
- ❌ **단점**
    
    - 배선 많음 (장치 늘릴수록 CS 핀 필요)
        
    - 장거리 통신엔 부적합 (노이즈 영향)
        
    - 에러 검출 기능 없음 (상위 레벨에서 처리 필요)
        

---

### 3. [[I2C(Inter-Intergrated Circuit)]]

- ✅ **장점**
    
    - **2선만으로 다중 장치 연결 가능** (주소 기반)
        
    - 멀티마스터 지원
        
    - ACK/NACK으로 기본적인 에러 검출 가능
        
    - 확장성 좋음 (센서, EEPROM 등 표준 지원 많음)
        
- ❌ **단점**
    
    - 속도 느림 (수백 kbps ~ 3.4 Mbps)
        
    - 풀업 저항 필요, 회로 복잡
        
    - 반이중이라 전송 지연 발생
        
    - 노이즈에 민감, 장거리 불리
        

---

# 📌 요약

- **UART** → 간단, 1:1 통신, 디버깅/저속 장치용
    
- **SPI** → 고속, 실시간 필요 장치(센서·메모리·LCD)
    
- **I²C** → 저속, 다중 센서 연결, 확장성 좋은 환경
    

---

👉 정리하면,

- **빠른 데이터 필요**: SPI
    
- **센서 여러 개, 간단 연결**: I²C
    
- **간단 1:1 연결, 디버깅**: UART
