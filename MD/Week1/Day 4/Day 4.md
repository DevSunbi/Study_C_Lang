## 배열
#### 배열의 선언
* 배열의 Type에 따라 배열의 공간이 결정된다
* C언어는 자체적으로 array out of index 체크 안 함
* ary\[n] => 주소와 동격 처리
* scanf를 통한 배열 요소 입력 시 & 사용 
	* 주소 전달을 통해 해당 주소에 직접 접근을 함
* Stack 공간에 배열 할당 됨 -> 자동으로 초기화 되지 않으며, 할당 시 자동으로 생성(junk 존재 가능)
	* 따라서 선언 시 초기화 선언 필수
	* 요소 수 < 초기 값 => 남은 요소는 Null로 초기화
	* 요소 수 > 초기 값 => Overflow 발생 (타 데이터 공간 침범 가능성 존재)
* 배열 초기화 시 요소의 수를 생략 가능함
	* 반드시 초기화에 의해 Size를 계산 할 수 있어야 함
* 선언한 자료형에 맞는 값으로 초기화
* 배열 타입, 주소, 크기 -> 같이 던져주는 것이 가장 확실한 방법
	* 배열 전체의 크기를 몰라도 배열의 기본 원소의 크기는 알 수 있다
	* 따라서 배열 전체의 크기 역산 가능
		* 전체 사이즈 / 요소 하나의 크기 => 배열의 크기(index 수) 알 수 있음
		* 배열 시작 주소만 넘겨줄 때 써먹기 좋음
		* count를 통해 loof를 안전하게 돌려먹기 좋음
#### 문자를 저장하는 배열
* 문자 배열은 초기화 시 \0 고려하며 요소 할당해야함(문자열과 비슷하게 취급)
	* -> n+1 만큼의 사이즈를 할당해야함
* 문자열 읽는 함수에서 enter키로 인해 문제 발생 빈번
	* newline char이 CRLF 2byte 표현하지만 Linux/Mac은 LF(1 byte) 방식
		* OS에 따라 읽는 방식이 다름
		* file system 다룰 때 fopen시 r, w사용 -> 런타임 라이브러리가 OS 환경에 맞게 newline character를 알아서 처리
		* 또는 \r 제거 로직을 통해 해결
* 초기화 방법
	* 문자 상수로 하나씩 초기화
	* 문자열 상수로 한 번에 초기화
	* 널 문자를 저장할 공간이 있어야 함
* scanf 함수 입력 시 배열 명만 사용하도록(scanf는 입력과 동시에 변환 함수다)
	* 문자열에서 애지간하면 gets, puts 사용하기
* puts func -> 공백도 데이터로 취급, newline character까지 읽어들인다
```C
int main(void) {
	char str[80];
	
	printf("문자열 입력 : ");
	gets(str);
	puts("입력된 문자열 : ");
	puts(str);
}
``` 
=> \n도 읽음을 생각
* 배열 사이즈보다 적게 요소 입력 후 배열 전체 출력
	* Null terminated Character 이전까지 출력(garbage 출력 가능성 존재)
* 배열에 값을 넣는다 -> 값을 복사하는 개념임(일반적인 요소는 깊은 복사, 포인터는 얕은 복사)
	* 포인터/구조체는 주소만 복사 되므로, 실 내용 복사를 위해선 malloc을 사용함
